1) Browser รู้ได้อย่างไรว่าต้องรัน JavaScript ใน <script> เมื่อโหลดหน้าเว็บ?
→ (อธิบาย 3–5 บรรทัด จากความเข้าใจของกลุ่ม)

ปกติแล้วเบราว์เซอร์จะอ่านโค้ด HTML เพื่อสร้าง DOM Tree ไปเรื่อย ๆ แต่พอไปเจอแท็ก <script> ปุ๊บ การสร้าง DOM จะ หยุดชะงัก (Block) ทันที เบราว์เซอร์จะไปจัดการโหลดไฟล์ JavaScript (ถ้ามี) แล้ว รัน โค้ดนั้นให้จบก่อน เพื่อให้แน่ใจว่า JS มีโอกาสแก้ไขหรือสร้าง DOM ขึ้นมาก่อนที่โครงสร้างหน้าเว็บที่เหลือจะถูกสร้างเสร็จ พอรันเสร็จถึงจะกลับไปสร้าง DOM ต่อครับ

2) ถ้าไม่มี DevTools การแก้ไขปัญหา (debug) เวลาเว็บมีปัญหาจะยากขึ้นอย่างไร?
→ (ตอบเป็น bullet list สั้น ๆ อย่างน้อย 3 ข้อ)

ตาบอดสนิทเรื่อง CSS/Layout: เราจะไม่สามารถ Inspect ดูโครงสร้าง HTML และ CSS ที่ถูกใช้งานจริงได้เลย ต้องเดาเอาว่าทำไม Layout ถึงพัง

ไม่เห็น Error Log: เวลา JavaScript มีปัญหา เราจะไม่เห็น Error Message หรือ Stack Trace เลย ทำให้การหาต้นตอของบั๊กเป็นไปได้ยากมาก ๆ ต้องพึ่ง alert() หรือ console.log() (ถ้ามี) แบบมั่ว ๆ

เช็คการเชื่อมต่อ API ไม่ได้: เราจะไม่รู้ว่าการเรียก API (Network Request) สำเร็จไหม, ข้อมูลที่ส่งไป/กลับมีหน้าตาเป็นอย่างไร หรือโหลดนานแค่ไหน ทำให้แก้ไขปัญหาที่เกี่ยวกับ Back-end ยากขึ้นมาก

ตาม Flow โค้ดไม่ได้: ไม่สามารถตั้ง Breakpoint เพื่อหยุดการทำงานของ JavaScript แล้วไล่ดูค่าตัวแปรหรือ Flow ของโปรแกรมแบบ Step-by-step ได้

3) แต่ละคนคิดว่า Panel ไหนใน DevTools จะได้ใช้บ่อยที่สุดในเทอมนี้ (Elements / Console / Network / อื่น ๆ)? เพราะอะไร?
→ ให้สมาชิกแต่ละคนตอบสั้น ๆ คนละ 1–2 บรรทัด พร้อมระบุชื่อในวงเล็บ เช่น

(จักรกฤษณ์) ชอบ Console เพราะใช้ดู error message, ดู log ที่เราตั้งไว้ และใช้ทดลองรันโค้ดเล็ก ๆ น้อย ๆ ได้ตลอดเวลา

(เมย์คาร์) ชอบ Elements ที่สุด เพราะเป็น Panel ที่ใช้ในการปรับแก้/ดีบัก CSS และการจัด Layout เพื่อให้หน้าเว็บออกมาเป๊ะตามที่เราต้องการ

(จิรวัฒน์) ชอบ Network ครับ เพราะเทอมนี้ต้องเรียก API เยอะมาก ต้องใช้มันเช็คสถานะการส่งข้อมูล, Payload และ Response ที่ได้กลับมา

(ปรเมษฐ) ผมว่า Sources ก็สำคัญนะ เพราะเวลาโค้ดมันซับซ้อนมากๆ การตั้ง Breakpoint เพื่อไล่ Flow ทีละบรรทัดช่วยชีวิตได้ดีกว่าการ console.log() เยอะเลยครับ

(ณัฐกิตติ์) ผมชอบ Elements เหมือนกันครับ แต่ใช้เพื่อดู DOM Tree เป็นหลัก เพื่อเช็คว่า JavaScript ที่เขียนไปมันเพิ่มหรือลบ Element ตามที่เราตั้งใจไว้ไหม
